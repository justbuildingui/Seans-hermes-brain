# Always Want the Simple

> Sean's stated operating preference: default to the simple option; complex/paid tooling comes later, only when the foundation is ready.

## State
- **Principle (exact phrasing):** "You should know that I always want the simple. so we'll use xurl later on once the brain is ready for that step"
- **Context originated:** GBrain install conversation with Ollie, 2026-04-20, while deciding between the heavy `x-to-brain` recipe (~$200/mo social monitoring) and `xurl` (simple auth'd curl wrapper) for tweet ingestion.
- **How it manifests:** Lead with the simple option in proposals. Flag complex/paid alternatives but don't default to them. Sequence work so simple lands first and complex only layers on once the substrate is ready.

## Implications
- When presenting options, order them simple → complex, not the other way.
- Paid tooling ($ >0) is a flag requiring explicit sign-off, not a default.
- "Ready for that step" implies a foundation gate — don't skip ahead.

## Open Threads
- Encode as a skill-level convention so every planning loop checks this before recommending paid/complex tools.
